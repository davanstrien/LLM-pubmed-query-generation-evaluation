# pubmed-query-generation-evaluation

This repository contains the code and data for evaluating the performance of various language models in assisting in generating PubMed search queries.
The evaluation is conducted on a set of prompts related to constructing PubMed queries for specific research topics.
These prompts are intended to be the sorts of queries that a librarian or researcher might use to prompt an LLM for help in constructing a search query.

For each prompt, the generated we pass the generated query to the PubMed API and evaluate the search results.
The current results include:

- the number of search results returned by PubMed **note** it should **not** be read that more is better, as the number of search results returned by PubMed is not a direct measure of the quality of the query. In many cases, too many results indicate a poorly constructed query that is too broad.
- the "Quoted Phrases Not Found Count": this indicates where a term uses in the search is not recognized as a valid MeSH term or keyword by PubMed. This can be seen as a very rough proxy for model "hallucination".
- Failed: whether the search query generated by the model caused an error when passed to the PubMed API. 
- Regex: whether the search query generated by the model required basic regex processing to be valid for the PubMed API. We aimed to make the prompts return queries that could be directly used in the PubMed API, but some models required additional processing to be valid.
- the search query generated by the model
